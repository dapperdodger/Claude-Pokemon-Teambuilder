# Teams

Built teams live here, one file per team. Copy `_TEMPLATE.md` to start a
new one — use a descriptive filename (e.g. `whimsicott-gholdengo-core.md`),
not a generic one like `team1.md`.

This isn't just a place to store the final six — the reasoning per pick
and the *intentional exclusions* matter as much as the roster itself, per
`CLAUDE.md` rule 11. A team file with no reasoning is just a Smogon-paste
copy with extra steps, and defeats the point of this whole repo.

Update a team's file in place as it evolves (with a changelog row, same
convention as `reference/`) rather than creating a new file per iteration.
